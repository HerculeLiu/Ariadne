# HTML 学习页参考样例（Baseline v1）

记录时间：2026-03-03  
用途：作为后续 UI/交互优化的对照基线。

## 目标
- 生成后直接进入完整学习页（不是前端容器里的 iframe 查看器）。
- Markdown 仅作为知识源，不直接按原样展示。
- 页面提供可操作学习流程（导航、进度、交互示例、巩固）。

## 当前基线入口
- 首页生成后跳转：`/downloads/{courseware_id}.html`
- 学习页生成函数：`src/ariadne/application/knowledge.py` 中 `markdown_to_html`

## 基线包含的关键体验
- 章节导航（左侧）
- Chunk 学习卡（主区）
- 学习进度条与上一块/下一块
- 交互示例运行（interaction block）
- 抽认卡显示/隐藏答案
- 固定返回首页按钮（右上角，路径 `/`）

## 基线质量标准（后续改版至少不退化）
- 不出现“原样 Markdown 查看器”体验。
- 页面主流程是“学习应用”而不是“文档阅读器”。
- 正文可读（背景与文字高对比）。
- `**`、`` ` ``、`#` 等 markdown 符号不应裸露影响阅读。
- 移动端可正常浏览和点击。
- 绝对禁止加入复习/考试/测试/练习类模块（quiz/flashcard/score/checkpoint）。

## 后续优化建议（按优先级）
1. Chunk 卡片多模板（概念型/对比型/实操型）自动切换
2. 交互示例支持“输入可编辑 + 输出模拟延迟”
3. 学习进度持久化（localStorage 或后端）
4. 章节小测（2-3 题）与错题回看
5. 视觉主题可切换（保持高对比）
