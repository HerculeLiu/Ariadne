# Ariadne 本地文件持久化方案 v1.0

## 1. 目标

本文档定义 Ariadne 在不引入数据库的前提下，如何使用本地文件系统完成以下对象的稳定持久化：

- `courseware`
- `chunk`
- `page`
- `chat session`
- `chat message`
- `asset`
- `snapshot/version`
- `generation job`（阶段二）

目标不是“把所有对象塞进一个大 JSON”，而是建立一套可追溯、可迁移、可恢复、可调试的文件持久化方案。

## 2. 设计原则

### 2.1 一个聚合对象一个目录

不要使用：

- `storage/chats.json`
- `storage/coursewares.json`（存全部详情）
- `storage/chunks.json`

采用：

- 一个 `courseware` 一个目录
- 一个 `asset` 一个目录
- 一个 `chat session` 一个文件
- 一个 `chunk` 一个文件
- 一个 `page` 一个目录

### 2.2 主真相单一

每类对象必须有唯一主真相来源：

- `courseware meta`：`coursewares/<cw_id>/meta.json`
- `chunk`：`coursewares/<cw_id>/chunks/<chunk_id>.json`
- `page`：`coursewares/<cw_id>/pages/<page_id>/meta.json`
- `chat session + messages`：`coursewares/<cw_id>/chats/<session_id>.json`
- `asset`：`assets/<asset_id>/meta.json`

禁止再出现以下情况：

- 一部分从内存 repo 推断
- 一部分从 markdown 反推
- 一部分从 html 反推
- 一部分从 `.meta.json` 猜测

### 2.3 文件系统负责对象与状态，向量库只负责检索

向量库存储：

- fragment embedding
- fragment metadata

向量库不负责：

- 课件结构
- chunk 正文主数据
- 聊天历史
- 页面定义

### 2.4 小文件优先，避免全量重写

支持高频修改的对象应拆成小文件：

- `chunk`
- `chat session`
- `asset`

避免每次改一个 chunk 都重写整份课件大 JSON。

### 2.5 原子写入

所有 JSON/Markdown/HTML 的写入必须采用原子写策略：

1. 先写入 `*.tmp`
2. 再使用 `rename` / `replace`
3. 成功后替换正式文件

避免进程中断留下半截文件。

## 3. 当前现状

### 3.1 已持久化

- `Asset`：`storage/assets/assets.json`
- `Knowledge markdown`：`storage/knowledge/<courseware_id>.md`
- `Knowledge metadata`：`storage/knowledge/<courseware_id>.meta.json`
- 向量片段：由向量库持久化

### 3.2 仍主要依赖内存

- `Courseware`
- `Chunk`
- `ChatSession`
- `ChatMessage`
- `GenerationJob`
- `RewriteDraft`
- `Page`

### 3.3 当前主要问题

- 服务重启后，`courseware/chat/chunk/page` 恢复依赖推断逻辑，不稳定
- `page_id` 存在于接口与聊天模型，但没有真实持久化实体
- `chunk` 状态与正文没有稳定持久化边界
- 聊天记录是内存态，重启即丢失
- `KnowledgeDocStore` 只解决了 markdown 和 `source_asset_ids`，没有承担完整对象存储职责

## 4. 目标目录结构

建议使用如下目录布局：

```text
storage/
  assets/
    as_xxx/
      source.bin
      meta.json
      fragments.json
  coursewares/
    cw_xxx/
      meta.json
      outline.json
      markdown.md
      html.html
      chunks/
        ck_001.json
        ck_002.json
      pages/
        pg_generated/
          meta.json
          html.html
      chats/
        sess_xxx.json
        sess_yyy.json
      snapshots/
        snap_001/
          meta.json
          markdown.md
          html.html
          chunks/
            ck_001.json
  indexes/
    assets.json
    coursewares.json
```

说明：

- `assets/`：按资产分桶
- `coursewares/`：按课件分桶
- `indexes/`：仅保留轻量索引，不存完整对象
- `snapshots/`：用于版本快照、rewrite、undo（阶段二）

## 5. 对象文件规范

## 5.1 Asset

路径：

```text
storage/assets/<asset_id>/
  source.bin
  meta.json
  fragments.json
```

### `meta.json`

```json
{
  "id": "as_845fe141",
  "file_name": "Hercule_Liu_Resume.pdf",
  "file_type": "pdf",
  "size_bytes": 123456,
  "status": "ready",
  "progress": 100,
  "error": null,
  "storage_path": "storage/assets/as_845fe141/source.bin",
  "content_preview": "...",
  "chunk_count": 18,
  "created_at": "2026-03-07T12:00:00Z",
  "updated_at": "2026-03-07T12:00:10Z"
}
```

### `fragments.json`

```json
[
  {
    "fragment_id": "frag_001",
    "page": 1,
    "offset": 0,
    "text": "..."
  }
]
```

说明：

- `fragments.json` 存可追溯文本切分结果
- embedding 向量仍由向量库存储

## 5.2 Courseware

路径：

```text
storage/coursewares/<courseware_id>/
  meta.json
  outline.json
  markdown.md
  html.html
```

### `meta.json`

```json
{
  "id": "cw_5d85a1bd",
  "topic": "AIPM 与简历项目经历",
  "status": "ready",
  "current_version": 1,
  "source_asset_ids": ["as_1", "as_2"],
  "markdown_path": "storage/coursewares/cw_5d85a1bd/markdown.md",
  "html_path": "storage/coursewares/cw_5d85a1bd/html.html",
  "default_page_id": "pg_generated",
  "created_at": "2026-03-07T12:00:00Z",
  "updated_at": "2026-03-07T12:05:00Z"
}
```

### `outline.json`

```json
[
  {
    "chapter_no": 1,
    "chapter_title": "AIPM 项目经历",
    "chunk_titles": [
      "项目背景",
      "核心职责",
      "结果与影响"
    ]
  }
]
```

说明：

- `outline.json` 独立存储，便于首页结构预览和调试
- `markdown.md` 与 `html.html` 是渲染产物，不再承担对象索引职责

## 5.3 Chunk

路径：

```text
storage/coursewares/<courseware_id>/chunks/<chunk_id>.json
```

### `chunk.json`

```json
{
  "id": "ck_001",
  "courseware_id": "cw_5d85a1bd",
  "page_id": "pg_generated",
  "chapter_no": 1,
  "chunk_no": 1,
  "order_no": 1,
  "title": "项目背景",
  "content": "...",
  "understand_state": "unknown",
  "is_favorite": false,
  "collapsed": false,
  "created_at": "2026-03-07T12:00:00Z",
  "updated_at": "2026-03-07T12:05:00Z"
}
```

说明：

- `chunk` 内容与 UI 状态先合并存储，降低第一阶段改造成本
- 后续如果状态更新频繁，可以拆分出 `chunk_state.json`

## 5.4 Page

路径：

```text
storage/coursewares/<courseware_id>/pages/<page_id>/
  meta.json
  html.html
```

### `meta.json`

```json
{
  "id": "pg_generated",
  "courseware_id": "cw_5d85a1bd",
  "page_type": "generated_shell",
  "title": "默认课件页面",
  "html_path": "storage/coursewares/cw_5d85a1bd/pages/pg_generated/html.html",
  "render_config": {
    "layout_mode": "continuous",
    "nav_collapsible": true
  },
  "created_at": "2026-03-07T12:00:00Z",
  "updated_at": "2026-03-07T12:05:00Z"
}
```

说明：

- `page_id` 必须有真实落点，不再只是字符串占位
- `page` 是聊天挂载点之一

## 5.5 Chat Session

路径：

```text
storage/coursewares/<courseware_id>/chats/<session_id>.json
```

### `session.json`

```json
{
  "id": "sess_abc123",
  "courseware_id": "cw_5d85a1bd",
  "page_id": "pg_generated",
  "chunk_id": "",
  "title": "关于 AIPM 项目经历的提问",
  "created_at": "2026-03-07T12:10:00Z",
  "last_active_at": "2026-03-07T12:12:00Z",
  "messages": [
    {
      "id": "msg_1",
      "role": "user",
      "content": "AIPM 项目做了什么？",
      "selected_context": "[Chapter 1 · Chunk 1]\n...",
      "asset_ids": ["as_1", "as_2"],
      "created_at": "2026-03-07T12:10:01Z"
    },
    {
      "id": "msg_2",
      "role": "assistant",
      "content": "...",
      "sources": [
        {"asset_id": "as_1", "fragment_id": "frag_002"}
      ],
      "created_at": "2026-03-07T12:10:05Z"
    }
  ]
}
```

说明：

- 一个 `session` 一个文件，适合 append-only 写入模型
- `selected_context` 必须持久化，否则历史回放不完整
- `asset_ids` 必须持久化，否则无法追溯问答使用了哪批文件

## 5.6 Snapshot / Version

路径：

```text
storage/coursewares/<courseware_id>/snapshots/<snapshot_id>/
  meta.json
  markdown.md
  html.html
  chunks/
```

### `meta.json`

```json
{
  "id": "snap_001",
  "courseware_id": "cw_5d85a1bd",
  "version": 1,
  "reason": "initial_generation",
  "created_at": "2026-03-07T12:05:00Z"
}
```

说明：

- 该目录用于 rewrite、undo、版本回滚
- 第一阶段可以先不实现，文档先定义

## 6. 轻量索引文件

## 6.1 `storage/indexes/coursewares.json`

```json
[
  {
    "id": "cw_5d85a1bd",
    "topic": "AIPM 与简历项目经历",
    "status": "ready",
    "updated_at": "2026-03-07T12:05:00Z",
    "path": "storage/coursewares/cw_5d85a1bd/meta.json"
  }
]
```

## 6.2 `storage/indexes/assets.json`

```json
[
  {
    "id": "as_845fe141",
    "file_name": "Hercule_Liu_Resume.pdf",
    "status": "ready",
    "updated_at": "2026-03-07T12:00:10Z",
    "path": "storage/assets/as_845fe141/meta.json"
  }
]
```

约束：

- 索引文件只保留摘要字段
- 不存完整聊天、不存完整 chunk、不存完整 markdown
- 首页列表优先读取索引，再按需加载对象详情

## 7. Repo 层改造建议

建议新增文件仓储实现，逐步替换现有内存 repo：

- `FileCoursewareRepo`
- `FileChunkRepo`
- `FilePageRepo`
- `FileChatSessionRepo`
- `FileAssetRepo`
- `FileSnapshotRepo`（阶段二）

保持现有 service 接口尽量不变，先通过 repo 注入替换底层持久化。

### 7.1 推荐职责划分

#### `FileCoursewareRepo`

负责：

- 读写 `coursewares/<id>/meta.json`
- 读写 `outline.json`
- 读写 `markdown.md`
- 读写 `html.html`
- 更新 `indexes/coursewares.json`

#### `FileChunkRepo`

负责：

- 读写 `chunks/*.json`
- 按 `courseware_id` 列出 chunk
- 按 `page_id` 或 `chapter_no` 过滤 chunk

#### `FilePageRepo`

负责：

- 读写 `pages/<page_id>/meta.json`
- 读写 `pages/<page_id>/html.html`

#### `FileChatSessionRepo`

负责：

- 读写 `chats/<session_id>.json`
- 列出某课件的所有会话
- 追加消息
- 更新 `last_active_at`

#### `FileAssetRepo`

负责：

- 读写 `assets/<asset_id>/meta.json`
- 维护 `indexes/assets.json`

## 8. 写入一致性规则

### 8.1 课件生成成功后

必须保证一次生成结束后以下文件状态一致：

1. `courseware/meta.json`
2. `courseware/outline.json`
3. `courseware/markdown.md`
4. `courseware/html.html`
5. `courseware/chunks/*.json`
6. `courseware/pages/pg_generated/meta.json`
7. `courseware/pages/pg_generated/html.html`
8. `indexes/coursewares.json`

推荐顺序：

1. 写 chunk 文件
2. 写 markdown/html
3. 写 page
4. 写 courseware meta
5. 最后更新 index

原则：

- index 最后写
- index 视为“对象可见性开关”

### 8.2 聊天消息写入

推荐顺序：

1. 读取 session 文件
2. 在内存中 append message
3. 原子写回同一 session 文件
4. 更新 `last_active_at`

### 8.3 删除与重建

- 删除 `courseware` 时，应删除整个 `coursewares/<id>/` 目录
- 删除 `asset` 时，应删除整个 `assets/<id>/` 目录
- 删除后同步更新索引文件

## 9. 迁移策略

## 9.1 第一阶段（建议立即实施）

目标：解决用户直接感知最强的丢失问题。

实施对象：

- `courseware`
- `chunk`
- `page`
- `chat session`
- `chat message`

实施结果：

- 服务重启后课件、chunk、页面、聊天仍可恢复
- `page_id` 成为真实持久化实体
- 聊天链路不再依赖纯内存 repo

### 第一阶段不改动

- 向量库
- `generation_job` 完整持久化
- `rewrite_draft`
- snapshot/version

## 9.2 第二阶段

实施对象：

- `generation job`
- `rewrite draft`
- `snapshot/version`

目标：

- 生成进度可恢复
- rewrite 支持可追溯版本回滚

## 9.3 第三阶段

实施对象：

- 老 `storage/knowledge/*.meta.json` 清理
- 老 `assets/assets.json` 迁移为目录式资产存储
- 索引重建命令

## 10. 与现有代码的衔接建议

### 10.1 `KnowledgeDocStore` 的角色调整

当前 `KnowledgeDocStore` 只管理：

- `<courseware_id>.md`
- `<courseware_id>.meta.json`

调整后建议：

- 将其角色缩小为 `CoursewareContentStore` 或并入 `FileCoursewareRepo`
- 不再承担“临时 metadata 补丁”职责
- 由正式 `courseware/meta.json` 管理 `source_asset_ids`

### 10.2 `CoursewareService.get()`

当前是：

- 先查内存 repo
- miss 时从 markdown 重建

调整后建议：

- 直接从 `FileCoursewareRepo` 读取 `meta + chunks + page`
- markdown 重建逻辑仅保留为迁移期 fallback

### 10.3 `ChatService`

调整后建议：

- `session` 与 `messages` 都从 `FileChatSessionRepo` 读取
- 每次回复后立即原子写回 session 文件

## 11. 风险与控制

### 风险 1：目录过多，小文件过多

结论：可接受。

原因：

- 当前场景是单机本地项目
- 可读性与可恢复性优先于极端吞吐量

### 风险 2：并发写入冲突

控制方式：

- 单进程先行
- 所有对象写入统一走 repo
- 使用原子写

### 风险 3：索引与对象不一致

控制方式：

- index 最后写
- 提供 `rebuild_indexes` 工具，从目录扫描重建索引

## 12. 验收标准

第一阶段完成后，应满足：

1. 服务重启后仍能打开既有课件页面
2. 服务重启后仍能读取既有 chunk 列表与内容
3. 服务重启后既有聊天记录仍可恢复
4. `page_id` 能查到真实 page 对象
5. 新生成课件的 `source_asset_ids` 可从 `courseware/meta.json` 恢复
6. 文件结构可人工检查，不依赖内存推断

## 13. 建议的落地顺序

1. 建立目录规范与原子写工具
2. 实现 `FileCoursewareRepo`
3. 实现 `FileChunkRepo`
4. 实现 `FilePageRepo`
5. 实现 `FileChatSessionRepo`
6. 修改 `build_services()` 注入新 repo
7. 保留 markdown 重建逻辑作为迁移期 fallback
8. 增加 `rebuild_indexes` 脚本

## 14. 最终结论

Ariadne 当前不必引入数据库。

更适合的方案是：

- 文件系统作为主持久化层
- 课件按目录隔离
- chunk/page/chat/session 按对象分文件
- 只保留轻量索引文件
- 向量库继续只负责检索

这条路线与 opencode 的本地持久化思想一致，但更适配 Ariadne 当前的对象模型和页面/课件/聊天三条主链路。
