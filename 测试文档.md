# Ariadne 测试文档（分阶段边界版）

## 1. 文档目标
按研发阶段拆分测试范围，每个阶段独立定义：
- 阶段目标
- 边界条件
- 必测用例
- 通过标准
- 放行门槛

对应开发阶段：
- PI-1（核心闭环，P0）
- PI-2（可用性增强，P1）
- PI-3（进阶能力，P2/P3）

## 2. 全局统一边界（适用于所有阶段）
- 输入校验：所有接口必须做字段类型、长度、枚举合法性校验。
- 状态机合法性：禁止非法状态跳转，必须记录错误日志与 trace_id。
- 并发一致性：涉及编辑与版本操作必须有版本冲突检测。
- 可观测性：关键动作必须产生日志与性能指标。
- 安全底线：上传文件类型白名单、输出内容转义、防注入。

## 3. PI-1 测试部分（核心闭环）

### 3.1 阶段目标
验证 P0 主链路稳定可用：
`输入主题 -> 生成课件 -> chunk 提问 -> 上传资料 -> 导出 HTML -> 未登录可用`

### 3.2 范围（用户故事）
- E1：US001、US007、US009
- E2：US015
- E3：US021
- E4：US031、US036
- E7：US061、US062
- E10：US091

### 3.3 PI-1 边界定义

#### 3.3.1 输入边界
- topic 长度：0、1、2、120、121。
- 关键词数量：0、1、5、10、11。
- 上传格式：PDF/MD/TXT 允许，其他拒绝。
- 上传大小：0B、上限、上限+1B。

#### 3.3.2 状态边界
- 生成状态：`queued -> retrieving -> generating -> assembling -> done/failed`。
- 解析状态：`queued -> processing -> ready/failed`。
- 非法状态跳转必须拒绝。

#### 3.3.3 时序边界
- 10 秒内出现“生成中”。
- 首屏结果 <= 15 秒（P95 <= 22 秒）。
- “聊这个点”点击后右侧侧边栏在 1 秒内打开。

#### 3.3.4 离线与权限边界
- 未登录状态可直接进入主题输入页并使用核心功能。
- 网络断开后不允许新生成请求，但已有本地课件可读。

### 3.4 PI-1 必测用例（阻断级）
- `TC-PI1-001`：topic 为空，提交按钮禁用。
- `TC-PI1-002`：topic=2 字符可提交并返回任务。
- `TC-PI1-003`：生成 10 秒内有进度事件。
- `TC-PI1-004`：生成成功后至少返回 3 个 chunk。
- `TC-PI1-005`：chunk “聊这个点”自动附带 chunk_id。
- `TC-PI1-006`：上传 PDF 成功进入可检索状态。
- `TC-PI1-007`：上传非法文件格式被拒绝。
- `TC-PI1-008`：导出 HTML 后离线可打开。
- `TC-PI1-009`：未登录可完成主链路。

### 3.5 PI-1 通过标准
- 阻断级用例通过率 100%。
- P0 故事验收通过率 100%。
- 无 S0/S1 缺陷。

### 3.6 PI-1 放行门槛
- 生成成功率 >= 97%。
- 导出成功率 >= 99%。
- 主链路 E2E 连续 3 次全通过。

## 4. PI-2 测试部分（可用性增强）

### 4.1 阶段目标
验证 P1 可用性和稳定性能力：
- 失败重试、参数恢复
- 多轮问答、引用来源、白名单检索
- 个性化设置与本地模式
- 日志、监控、单测覆盖、E2E 完整化

### 4.2 范围（用户故事）
- E1：US003、US005、US008、US010
- E2：US011、US016、US017、US018、US020
- E3：US022、US023、US024、US028
- E4：US032、US033、US035、US037、US038、US040
- E5：US041、US043、US044、US050
- E7：US063、US064、US065、US066、US068、US069、US070
- E9：US081、US086
- E10：US093、US094、US095、US096、US097、US098

### 4.3 PI-2 边界定义

#### 4.3.1 参数与配置边界
- difficulty/style/template 非法枚举值拒绝。
- 白名单域名：空、合法、重复、非法域名。
- 来源偏好权重总和必须为 1（或系统自动归一化）。

#### 4.3.2 会话上下文边界
- 多轮会话上下文窗口上限（如最近 N 轮）可控。
- 文本选区为空时禁止提问。
- 跨页编辑时目标页不存在应返回可读错误。

#### 4.3.3 数据一致性边界
- “补充到 chunk”必须原子写入：正文 + 版本 + 日志。
- AI 改写未确认不得落库。
- 撤销仅允许最近一次已应用修改。

#### 4.3.4 性能边界
- 常规生成首屏 <= 15 秒。
- 问答响应 P95 <= 5 秒。
- 目录跳转 <= 1 秒。

#### 4.3.5 离线边界
- 离线状态可查看已缓存课件、笔记。
- 网络恢复后可执行补同步。

### 4.4 PI-2 必测用例（阻断级）
- `TC-PI2-001`：生成失败后一键重试，参数与上次一致。
- `TC-PI2-002`：刷新后恢复最近一次生成参数。
- `TC-PI2-003`：选中文本提问自动附带选区内容。
- `TC-PI2-004`：回答展示关联 chunk 编号与核心术语。
- `TC-PI2-005`：采纳补充产生新版本，拒绝补充不改正文。
- `TC-PI2-006`：白名单限定检索结果域名 100% 命中。
- `TC-PI2-007`：解析失败返回可读错误并可重试。
- `TC-PI2-008`：本地模式开启后无远程记忆同步请求。
- `TC-PI2-009`：AI 改写先预览再确认，未确认不写入。
- `TC-PI2-010`：撤销最近修改后页面恢复前一版本。
- `TC-PI2-011`：日志可按时间筛选并含 trace_id。
- `TC-PI2-012`：E2E 覆盖输入-生成-导出链路。

### 4.5 PI-2 通过标准
- P1 范围用例通过率 >= 95%。
- 后端核心模块单测覆盖率 >= 70%。
- 性能指标达标（见 4.3.4）。

### 4.6 PI-2 放行门槛
- 无 S0/S1 缺陷。
- 关键回归集 100% 通过。
- 近 7 天错误趋势无显著上升（同口径对比）。

## 5. PI-3 测试部分（进阶能力）

### 5.1 阶段目标
验证 P2/P3 进阶能力在规模、复杂交互与长期使用场景下稳定：
- 图谱可视化
- 复习调度
- 资产高级能力
- 无障碍与国际化

### 5.2 范围（用户故事）
- E6：US051~US060
- E8：US071~US080
- E9：US082~US090（除已在 PI-2 覆盖项）
- E10：US092、US099、US100
- 以及其余未在 PI-1/PI-2 关闭的 P2/P3 项

### 5.3 PI-3 边界定义

#### 5.3.1 图谱边界
- 节点数量：10、100、300（100 内必须流畅）。
- 关系密度：稀疏图/稠密图。
- 筛选条件组合：学科+日期+掌握度。

#### 5.3.2 复习调度边界
- 卡片量：10、100、1000。
- 间隔重复参数极值（最短/最长间隔）。
- 连续中断学习后的补学建议生成。

#### 5.3.3 资产与版本边界
- 导入课件包格式兼容性（标准/损坏/缺字段）。
- 回滚到历史版本后保留当前快照。
- 标签数量上限与全局搜索性能。

#### 5.3.4 可访问性与国际化边界
- 键盘全流程可操作（Tab/Enter/Esc）。
- 中英切换即时生效，核心菜单文案无遗漏。

### 5.4 PI-3 必测用例（阻断级）
- `TC-PI3-001`：图谱 100 节点缩放拖拽流畅。
- `TC-PI3-002`：点击节点准确跳转到目标 chunk。
- `TC-PI3-003`：chunk 可生成复习卡片并更新复习日期。
- `TC-PI3-004`：导入标准课件包成功并可继续编辑。
- `TC-PI3-005`：版本回滚后保留回滚前快照。
- `TC-PI3-006`：键盘可完成主操作流程。
- `TC-PI3-007`：中英切换后核心页面文案全部生效。

### 5.5 PI-3 通过标准
- P2/P3 目标用例通过率 >= 90%。
- 大规模数据下无阻断级性能退化。
- 可访问性核心检查通过。

### 5.6 PI-3 放行门槛
- 无 S0/S1。
- 图谱与复习核心链路可稳定连续运行。
- 资产导入/导出/回滚闭环可用。

## 6. 跨阶段公共策略

### 6.1 缺陷分级
- `S0`：数据丢失、主链路不可用、安全事故（必须阻断）。
- `S1`：核心能力严重异常（原则阻断）。
- `S2`：有替代路径（评估放行）。
- `S3`：轻微问题（可带病发布）。

### 6.2 自动化覆盖策略
- 单测：参数校验、状态机、版本冲突、排序逻辑。
- 集成：接口契约、事务一致性、幂等复用。
- E2E：每阶段至少 1 条完整链路 + 1 条异常链路。

### 6.3 回归策略
- 每阶段结束进行全量回归。
- 下一阶段开始前执行上阶段阻断级冒烟。
- 跨阶段改动必须回归 PI-1 主链路。

## 7. 阶段交付物
每个阶段必须提交：
- 阶段测试计划
- 阶段测试用例集（含边界标签）
- 阶段测试报告（通过率、缺陷、风险）
- 放行建议（放行/不放行）
