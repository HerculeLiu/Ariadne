# Ariadne 接口文档（当前实现）

## 1. 文档说明

- 基础路径：`/api/v1`
- 响应格式：JSON（文件上传除外）
- 本文只记录 **当前代码已经实现** 的接口
- 未实现或已废弃的旧接口不再作为主文档描述

---

## 2. 统一响应格式

成功：

```json
{
  "code": 0,
  "message": "ok",
  "trace_id": "tr_xxx",
  "data": {}
}
```

失败：

```json
{
  "code": 10003,
  "message": "resource not found",
  "trace_id": "tr_xxx",
  "error": {
    "field": "courseware_id",
    "reason": "not found"
  }
}
```

## 4. 课件接口

### 4.1 生成课件

`POST /api/v1/coursewares/generate`

请求体：

```json
{
  "topic": "Transformer",
  "keywords": ["attention", "encoder", "decoder"],
  "asset_ids": ["as_001", "as_002"]
}
```

说明：
- `asset_ids` 可选
- 传入后会参与 outline 检索和 chunk 检索

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "courseware_id": "cw_xxx",
    "phase": "queued"
  }
}
```

### 4.2 查询生成进度

`GET /api/v1/coursewares/{courseware_id}/progress`

响应字段包含：
- `phase`
- `progress`
- `outline`
- `chunk_total`
- `chunk_done`
- `completed_chunks`

### 4.3 获取课件详情

`GET /api/v1/coursewares/{courseware_id}`

当前返回重点字段：
- `id`
- `topic`
- `status`
- `current_version`
- `source_asset_ids`
- `knowledge_doc_path`
- `knowledge_markdown`
- `chunks`

`chunks` 当前包含：
- `id`
- `title`
- `content`
- `order_no`
- `chapter_no`
- `chunk_no`
- `page_id`
- `understand_state`
- `is_favorite`
- `collapsed`

### 4.4 获取课件目录 / 内容

`GET /api/v1/coursewares/{courseware_id}/chunks`

### 4.5 获取 Markdown

`GET /api/v1/coursewares/{courseware_id}/markdown`

### 4.6 更新 Markdown

`PUT /api/v1/coursewares/{courseware_id}/markdown`

请求体：

```json
{
  "markdown": "# 标题\\n\\n## 第1章\\n...",
  "expected_version": 3
}
```

### 4.7 获取 HTML

`GET /api/v1/coursewares/{courseware_id}/html`

### 4.8 导出课件

`POST /api/v1/coursewares/{courseware_id}/export`

### 4.9 撤销最近修改

`POST /api/v1/coursewares/{courseware_id}/undo`

请求体：

```json
{
  "page_id": "pg_generated",
  "expected_version": 4
}
```

说明：
- 当前 undo 以 `courseware_id` 作为主定位参数
- `page_id` 仅用于上下文与日志，不再承担课件定位职责

## 5. Chunk 接口

### 5.1 更新学习状态

`PATCH /api/v1/chunks/{chunk_id}/state`

请求体：

```json
{
  "understand_state": "understood",
  "is_favorite": true,
  "collapsed": false
}
```

### 5.2 提问

`POST /api/v1/chunks/{chunk_id}/ask`

请求体：

```json
{
  "question": "这一段在讲什么？"
}
```

### 5.3 追加内容

`POST /api/v1/chunks/{chunk_id}/append`

### 5.4 应用 chunk 修改

`POST /api/v1/chunks/{chunk_id}/apply`

### 5.5 删除 chunk

`POST /api/v1/chunks/{chunk_id}/delete`

请求体：

```json
{
  "courseware_id": "cw_xxx",
  "expected_version": 3
}
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "version": 4,
    "deleted_chunk_id": "ck_xxx"
  }
}
```

## 6. 聊天接口

### 6.1 创建会话 / 读取现有会话

`POST /api/v1/chat/sessions`

创建会话请求：

```json
{
  "courseware_id": "cw_xxx",
  "page_id": "pg_generated"
}
```

读取现有会话请求：

```json
{
  "session_id": "cs_xxx"
}
```

### 6.2 获取会话列表

`GET /api/v1/chat/sessions?courseware_id=cw_xxx&page_id=pg_generated`

### 6.3 发送消息

`POST /api/v1/chat/messages`

请求体：

```json
{
  "session_id": "cs_xxx",
  "message": "古诗讲的是什么",
  "asset_ids": ["as_001"],
  "selected_chunk_ids": ["ck_001", "ck_002"],
  "selected_context": "[Chapter 1 · Chunk 1]\\n..."
}
```

说明：
- `message`：用户真实问题
- `asset_ids`：可选，未传时会 fallback 到 `courseware.source_asset_ids`
- `selected_chunk_ids`：对所选 chunk 做软加权
- `selected_context`：仅作为 fallback hint，不再是强制上下文

当前聊天检索架构：
1. 检索 `courseware chunks`
2. 检索 `source fragments`
3. 合并成统一 `rag_context`

## 11. 页面访问

虽然不是 `/api/v1` 接口，但当前系统还有一个重要页面路由：

- `GET /downloads/{courseware_id}.html`

用途：
- 导出后的 HTML 课件页面
- 也是当前课程阅读壳层的主要访问入口
